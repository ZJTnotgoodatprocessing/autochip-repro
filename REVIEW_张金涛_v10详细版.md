# 第一部分：语言、文字与写作风格

## 1.1 LLM 风格段落

### 1.1.1 §3.9 本章小结：从四个"通过"变成了五个"通过"（v10 退步点）

第一轮评审已明确批评 §3.9 的"四段式通过……排比"是 LLM 风格的典型标志。但 v10 (`chapter3.tex:206`) 改写后变成：

> 系统**通过**统一的 Task 接口屏蔽了两种 benchmark 的格式差异，**通过**两阶段提取策略处理了 LLM 的多样化回复格式，**通过** $[-2,1]$ 标量评分实现了编译仿真结果的统一量化，**通过**五级反馈粒度和多轮对话两种机制为反馈构造提供了灵活配置，**通过**元数据记录和审计脚本确保了实验结果的可追溯性。

四个变五个，问题被放大。答辩时几乎必然被注意到。

**修改建议**：把小结改成对设计 rationale 的提炼，而非对本章设计的逐条复述：

> 本章的核心设计决策是把"benchmark 差异"封装在加载器、"LLM 回复差异"封装在提取器，下游的执行、评分、反馈构造和实验管理只依赖 Task / Verilog 文本 / rank 三种解耦的中间产物。L0--L4 粒度与单/多轮模式能够零侵入接入主流程的原因正是这一解耦。

### 1.1.2 §3.1 五项工程挑战的排比

`chapter3.tex:10-18` 以"第一……第二……第三……第四……第五……"罗列五项挑战，每一条句式相似（"X 不统一/不可控/需要转化为/需要平衡/必须保证"）。这是 v10 新增结构，结构本身没问题，但 5 条同等并列的排比感强。

**修改建议**：把"benchmark 格式不统一"和"LLM 回复格式不可控"合并为一条（都属于外部异构输入的清洗问题），第五条作为独立的工程支撑铺垫，把 5 条压成 3 条主线。

### 1.1.3 §2.1.2 与 §2.2.3 的"X 层面"三段并列

- `chapter2.tex:16`："在**并行性**层面，……在**时钟与复位**层面，……在**位宽语义**层面，……"
- `chapter2.tex:32`："**语法层面**的错误（……）；**接口层面**的不匹配（……）；**功能层面**的边界条件错误（……）；以及**算法层面**的理解偏差（……）"

第一轮已在 §5.9 类似句式上批评过，第 2 章这两处仍未处理。

**修改建议**：§2.1.2 三层差异合并为一段自然叙述："Verilog 的并发语义、对时钟复位的依赖、对位宽的精确要求，三者共同决定了 RTL 不能像普通软件那样依赖纯文本生成。"

### 1.1.4 §5.5 "信息过载"段类比存疑（v10 新增问题）

`chapter5.tex:178`：

> 这一"信息过载"现象与**信息论中的有限注意力模型**一致。

**问题**："有限注意力 (bounded attention)"并非信息论的标准概念——这一术语主要属于认知心理学、行为经济学和神经科学。信息论的核心是信道容量、互信息、信源编码定理，并不存在通称的"有限注意力模型"。答辩时可能被追问出处。

**修改建议**：要么删除该类比，要么改为不依赖"信息论"框架的描述：

> 这一现象与注意力机制在长上下文中常见的信噪比下降问题一致：当反馈包含的非关键信息超出阈值后，模型对关键错误位置的定位反而被稀释。
