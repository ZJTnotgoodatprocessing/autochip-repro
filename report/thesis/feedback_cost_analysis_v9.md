# 反馈循环成本效率分析

> 生成日期：基于已有实验记录统计，未重跑实验

## 统计口径

- Zero-shot：每题 1 次 API 调用
- Feedback/Retry-only：每题最多 k × N 次调用（k=每轮候选数, N=最大迭代轮数）
- 实际调用次数 = 收敛轮数 × k（成功收敛后停止迭代）
- 数据来源：各实验 summary.json 中的 fb_iterations / ro_iterations 字段

## 成本效益结论

1. 实际调用次数远低于理论上限（15次/题），大部分成功题目在前1-2轮收敛
2. 高难度任务（RTLLM_STUDY_12）反馈循环性价比最高：+33pp 增益约需 3-5 倍调用
3. 简单任务（VerilogEval-Human 80%基线）反馈边际收益仅 +10pp，性价比较低
4. 建议策略：先 zero-shot，仅对失败题启用反馈循环
